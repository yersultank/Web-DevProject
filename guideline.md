# 📋 Project Guidelines

## 👥 Group Rules

- Exactly **3 students** per group
- All members must be from the **same practice group**
- Submit to your practice teacher **ASAP** (before pre-defense):
  - Group member names
  - Project name
  - GitHub repository link

> ⚠️ If group info is not submitted, you will **not be allowed to defend**.

---

## 🚀 GitHub Setup (Do This First)

- [ ] Create a GitHub repository
- [ ] Add a `README.md` with:
  - Project description
  - Team member names
- [ ] Initialize Angular project: `ng new project-name`
- [ ] Push initial code to the repo

---

## 🅰️ Frontend Requirements (Angular)

- [ ] Call APIs using **services + interfaces**
- [ ] At least **4 API-triggering events** (e.g. button clicks, page loads)
- [ ] At least **4 form inputs** using `ngModel`
- [ ] **Routing** with 3+ pages
- [ ] Basic **styling**
- [ ] Use `@for` / `@if` (or `*ngFor` / `*ngIf`)
- [ ] **JWT authentication**: login, logout, and HTTP interceptor
- [ ] Handle **API errors** gracefully

---

## 🐍 Backend Requirements (Django + DRF)

- [ ] At least **4 models**
- [ ] **2+ ForeignKey** relations between models
- [ ] Serializers: **2 regular Serializer** + **2 ModelSerializer**
- [ ] Views: **2 Function-Based Views (FBV)** + **2 Class-Based Views (CBV)**
- [ ] Token-based **auth**: login + logout
- [ ] **Full CRUD** for at least one model
- [ ] Link data to **`request.user`**
- [ ] Enable **CORS**
- [ ] Include a **Postman collection** in the repo

---

## 🎤 Defense (Week 14)

- **5–7 minutes** presentation + Q&A
- Max **4 slides** (PDF format)
- **Live demo** is required
- Be ready to explain **both frontend and backend**
- Bring a **working project on a laptop**
